Webhook Receiver

GitHub Webhook → Validation → Vercel Deploy Hook Trigger

Service Status

Webhook endpoint:
https://your-project-name.vercel.app/webhook
HTTP method:
POST
Content-Type:
application/json
Required header:
X-Hub-Signature-256
Supported events:
push, pull_request

Configuration Checklist

  1. Environment variables set in Vercel dashboard:
    • GITHUB_WEBHOOK_SECRET — same value as in GitHub webhook settings
    • VERCEL_DEPLOY_HOOK_URL — copied from Vercel → Settings → Git → Deploy Hooks
  2. GitHub webhook created with:
    • Payload URL: https://your-project-name.vercel.app/webhook
    • Content type: application/json
    • Secret: matches GITHUB_WEBHOOK_SECRET
    • Events: Pushes + Pull requests (opened, synchronize, reopened)
  3. Deploy Hook created in Vercel project settings

Request Flow

GitHub
→
Webhook /webhook
→
Signature validation
→
Event filter (main / PR)
→
Trigger Vercel Deploy Hook

Testing & Debugging

You can test the endpoint locally using:

curl -X POST http://localhost:3000/webhook \
  -H "Content-Type: application/json" \
  -H "X-Hub-Signature-256: sha256=..." \
  -d '{"ref":"refs/heads/main","repository":{"full_name":"owner/repo"}}'

Check Vercel function logs for:

  • Invalid signature warnings
  • Skipped events (branch/action not matching criteria)
  • Successful deployment trigger messages
